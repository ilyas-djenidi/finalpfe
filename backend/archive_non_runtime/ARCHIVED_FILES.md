# ARCHIVED FILES — ما نُقل إلى هذا المجلد ولماذا

ملفات وأسباب أرشفتها مؤقتاً هنا (يمكن حذفها نهائياً بعد التأكد):

- `arch_extract.txt` — ملاحظات معمارية ومدوّنة (وثائق)، ليست مطلوبة لتشغيل التطبيق.
- `DOCUMENTATION_COMPLETE_PROJET_CYBRAIN_FR.txt` — توثيق مُفصّل للمشروع باللغة الفرنسية، للمرجعية فقط.
- `sample/` — يحتوي ملفات عرض تقديمي (PPTX)، ملفات إعدادات HTTP قديمة، وملفات نصية استخراجية (`ppt_extracted.txt`, `Reponse_Exigences_Prof_FR.txt`). هذه مواد مرجعية/عرضية وليست ضرورية للتشغيل.

- `.env.example` — قالب متغيرات بيئية. احتفظنا بـ`.env` الفعلي في الجذر، والقالب مرجعي فقط.
- `httpd_cybrain_fixed14.conf` — نسخة إعدادات Apache قديمة/نسخ احتياطي غير مستخدمة مباشرةً أثناء التشغيل.

- `.env.example` — قالب متغيرات بيئية. احتفظنا بـ`.env` الفعلي في الجذر، والقالب مرجعي فقط.
- `httpd_cybrain_fixed14.conf` — نسخة إعدادات Apache قديمة/نسخ احتياطي غير مستخدمة مباشرةً أثناء التشغيل.
- `docker-compose.yml`, `Dockerfile` — إعدادات Docker، مخصصة للبيئة الإنتاجية فقط (لا تُستخدم محلياً في التطوير).
- `gunicorn.conf.py` — إعدادات gunicorn (خادم WSGI للإنتاج)، غير مطلوبة في التطوير المحلي.
- `*.bat`, `*.ps1` — سكريبتات بدء/إيقاف/إعادة تشغيل الخادم (مرجعية/توثيق فقط؛ التطبيق يُشغّل مباشرة عبر `python app.py`).
- `test_scanner.py` — اختبارات قديمة/غير مستخدمة; الاختبارات الفعلية في `tests/` (مُؤرشف أيضاً).
- `tests/` — مجلد الاختبارات (pytest أو اختبارات يدوية) — مرجعي للمطورين.

**لماذا نقلتُها:**
- لتحسين تنظيم المشروع وتقليل الضوضاء في جذر المشروع أثناء التطوير.
- الملفات المُؤرشفة قد تكون كبيرة (PPTX) أو متعددة الإصدارات (httpd_cybrain_fixed*.conf) وبالتالي تؤثر على قابلية التتبّع.
- فصل ملفات الإنتاج والتطوير يُحسّن الوضوح: جذر نظيف = ملفات تشغيل حية فقط.
